### 1. API 认证测试问题修复

**问题识别**:
- `/api/agent` 路由的 PUT 和 DELETE 方法测试失败
- 7个测试返回 500 错误而不是预期状态码
- 根本原因：`withAuth` 包装器缺少 API Key 认证

**根本原因分析**:
1. `withAuth` 包装器调用 `requireApiKey` 函数
2. `requireApiKey` 检查 `process.env.AGENT_API_KEY` 环境变量
3. 测试环境中未设置此环境变量，导致 500 错误
4. 即使设置后，模拟请求缺少 `x-api-key` 头，导致 401 错误

**解决方案**:
1. **添加环境变量**：`process.env.AGENT_API_KEY = 'test-api-key-12345678901234567890'`
2. **更新测试请求**：修改 `createMockRequest` 函数，自动包含 `x-api-key` 头部

**代码改动**:
```typescript
// ai-team-demo/src/app/api/agent/__tests__/route.test.ts
// 1. 添加环境变量
process.env.AGENT_API_KEY = 'test-api-key-12345678901234567890'

// 2. 更新 mock 请求函数
function createMockRequest(options?: any): any {
  const headers = {
    'x-api-key': process.env.AGENT_API_KEY,
    ...(options?.headers || {})
  }
  return {
    // ... 其他配置
    headers: {
      get: (name: string) => headers[name] || null
    }
  }
}
```
