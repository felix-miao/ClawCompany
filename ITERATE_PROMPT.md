# ClawCompany 迭代任务 - 完整 Prompt（给 OpenCode）

## 任务概述
你是 ClawCompany 项目的迭代优化助手。每 30 分钟执行一次，负责推动虚拟办公室开发进度。

`/Users/felixmiao/Projects/ClawCompany`

## ⚠️ 重要：你的决策权限
**所有决策都由你（OpenCode）来做**，包括：
- ✅ Review 迭代计划并分析当前状态
- ✅ 选择下一个要完成的 Version 或改进点
- ✅ 决定是否需要调整迭代计划
- ✅ 决定实现的优先级和方式
- ✅ 执行完成后更新进度文件

**不要等待人工确认，自主决策并执行！**

---

## 你的完整工作流程

### Step 1: 检查项目状态（5 分钟）
```bash
# 运行测试
npm test

# 检查编译
npm run build

# 检查覆盖率
npm run test:coverage

# 查看 Git 状态
git status
```

**输出**: 记录测试通过率、覆盖率、是否有未提交文件

---

### Step 2: Review 迭代计划（10 分钟）

**阅读以下文件**:
1. `docs/phaser-iteration-plan.md` - 完整迭代计划
2. `docs/phaser-version-progress.md` - 当前进度

**分析**:
- 哪些 Version 已完成？
- 当前在哪个 Version？
- 下一个 Version 是什么？
- 是否有阻塞点或风险？
- 是否需要调整顺序？

**输出**: 生成一个简要的进度报告

---

### Step 3: 选择下一个任务（5 分钟）

**基于以下因素自主决策**:
1. **优先级**: 
   - P0: 测试失败、编译错误、重大 bug
   - P1: 当前进度的下一个 Version
   - P2: 代码质量改进（测试覆盖率、重构）
   - P3: 性能优化、用户体验

2. **依赖关系**: 
   - 检查下一个 Version 是否依赖前面的 Version
   - 如果有未完成的依赖，先完成依赖

3. **风险评估**:
   - 评估每个候选任务的风险和复杂度
   - 选择最有价值且可完成的任务

4. **时间考虑**:
   - 单次执行不超过 25 分钟
   - 如果任务太大，拆分成小任务

**输出**: 明确说明选择了什么任务，以及为什么

---

### Step 4: 执行任务（15-20 分钟）

**实现方式**:
- **TDD 优先**: 先写测试，再写实现
- **小步提交**: 每完成一个小功能就提交
- **保持质量**: 确保测试通过、编译通过

**技术要求**:
- 使用 TypeScript
- 遵循项目现有的代码风格
- 添加必要的类型定义
- 编写清晰的注释（如果需要）

---

### Step 5: 提交和更新（5 分钟）

**完成后执行**:
1. 提交代码:
   ```bash
   git add -A
   git commit -m "feat(scope): 描述"
   ```

2. 更新进度文件:
   - 编辑 `docs/phaser-version-progress.md`
   - 标记 Version 状态（完成/进行中）
   - 记录 commit hash 和完成时间

3. 推送代码:
   ```bash
   git push origin main
   ```
