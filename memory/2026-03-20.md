### ✅ 创建 Commit 质量标准

**文件：** `COMMIT_STANDARD.md`

**核心规则：**
1. ✅ **允许的 Commit 类型：**
   - `feat` - 新功能/新模块（必须有可运行代码）
   - `fix` - Bug 修复（必须有测试验证）
   - `refactor` - 代码重构（必须改善质量）
   - `test` - 测试完善（必须提高覆盖率）
   - `perf` - 性能优化（必须有数据对比）

2. ❌ **禁止的 Commit 类型：**
   - `docs` - 纯文档更新（必须和代码一起提交）
   - `chore` - 简单配置/进度（禁止心跳检查类提交）
   - `style` - 代码格式（应该在功能开发中完成）

3. **每 2 小时的 Commit 任务：**
   - 任务 A：实现新功能模块（30-60分钟）
   - 任务 B：完善已有模块（30-60分钟）
   - 任务 C：完善测试用例（30-60分钟）

**研究成果:**

   - 完全支持创建 sub-agents
   - 支持两种 runtime: subagent (轻量级) 和 acp (编码代理)
