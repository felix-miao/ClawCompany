## ClawCompany Multi-Agent 系统

### 已完成的优化

#### 1. 数据共享软链接 ✅
- 所有 agent 的 `data/` 都指向 `~/.openclaw/shared-data/`
- 使用追加式更新，不覆盖
- SOUL.md 已更新：写入规则

#### 2. Telegram Token 安全 ✅
- 已创建 credentials 文件：
  - `~/.openclaw/credentials/telegram-sidekick.json`
  - `~/.openclaw/credentials/telegram-pm.json`
- 仍需配置环境变量或让 OpenClaw 从 credentials 读取

#### 3. Agent 间通信权限
- **决定**：保持全局 allow 列表（简化配置）
- **约束**：通过 SOUL.md 限制每个 agent 只能调用特定 agent

### 优化决策记录

| 问题 | 决策 | 理由 |
|------|------|------|
| agentToAgent 权限 | 保持现状 | OpenClaw 不支持逐级权限，全局已足够安全 |
| 数据共享 | 软链接 + 追加更新 | 简单高效，跨 agent 可见 |
| Token 存储 | 先迁移到 credentials， 中等优先级 | 不影响当前功能 |
| 任务监控 | 不实施 | 目前用文件轮询 + session 监控即可 |

### 关于任务执行状态监控

**结论： 不实施新的轮询机制。**

**原因：**
1. `sessions_list` + `sessions_history` 已经可以监控 agent 状态
2. 文件轮询 + session 监控 双重保障已经足够
3. 可以在 SOUL.md 中添加超时检测逻辑（agent 自行判断）

### 已确认的完整工作流

```
用户 (Telegram)
  ↓
sidekick (消息路由器)
  ↓ 调用 pm
  ↓
pm 分析需求， 写入 plan.md
  ↓ 调用 developer
  ↓
developer 实现， 执行测试
  ↓ 调用 tester
  ↓
tester 测试
  ↓ 调用 pm
  ↓
pm 调用 reviewer
  ↓
reviewer 审核
  ↓ 返回 sidekick
  ↓
sidekick 回复用户
```
