# OpenCode 监督指南

## 我的角色

### ✅ 我负责
1. **启动 OpenCode**
2. **监督执行**
3. **检查结果**

### ❌ 我不负责
1. **决策**（由 OpenCode 自己做）
2. **实现**（由 OpenCode 执行）
3. **编写代码**（由 OpenCode 写）

---

## 监督流程

### Step 1: 启动 OpenCode（2 分钟）

**正确启动方式**:
```typescript
exec({
  command: "cd ~/Projects/ClawCompany && opencode run '任务描述（来自 prompt 文件）'",
  pty: true,           // 必须！
  background: true,    // 后台运行
  timeout: 3600,       // 1 小时超时
  workdir: "/Users/felixmiao/Projects/ClawCompany"
})
```

**记录**:
- Session ID（用于监控）
- 启动时间
- 任务描述

---

### Step 2: 监督执行（每 5 分钟检查一次）

**使用 `process` 工具监控**:
```typescript
// 查看输出
process({ action: "log", sessionId: "session-id", limit: 50 })

// 检查是否还在运行
process({ action: "poll", sessionId: "session-id" })
```

**监督要点**:
1. ✅ OpenCode 是否在输出日志？（有进度）
2. ✅ 是否在执行文件操作？（Read/Write/Edit）
3. ✅ 是否在运行测试？（npm test）
4. ⚠️ 是否卡住了？（超过 10 分钟没有输出）
5. ⚠️ 是否报错了？（错误信息）

**异常处理**:
- **卡住超过 10 分钟** → 杀掉进程，重启
- **报错但未退出** → 等待 OpenCode 自己修复
- **提前退出** → 检查结果，如果失败则重启

---

### Step 3: 检查结果（任务完成后）

**检查清单**:
```bash
# 1. 检查 Git 提交
git log --oneline -1

# 2. 检查测试
npm test

# 3. 检查编译
npm run build

# 4. 检查进度文件
git diff docs/phaser-version-progress.md

# 5. 检查代码改动
git status
git diff --stat
```

**验证要点**:
1. ✅ 是否有新的 Git commit？
2. ✅ 测试是否全部通过？
3. ✅ TypeScript 是否编译成功？
4. ✅ 进度文件是否更新？
5. ✅ 改动是否符合任务描述？

---

## 监督标准

### ✅ 正常执行
- 有持续的日志输出
- 文件操作正常（Read/Write/Edit）
- 测试运行正常
- 最终有 Git commit
- 进度文件已更新

### ⚠️ 需要关注
- 输出变慢（超过 5 分钟没有新输出）
- 重复相同的操作（可能卡住）
- 测试失败但继续执行（可能在修复）
- 没有文件改动（可能在分析阶段）

### 🚨 需要干预
- 卡住超过 10 分钟（杀掉重启）
- 报错后未自动修复（重启或跳过）
- 提前退出但未完成任务（重启）
- 产生了破坏性改动（回滚并重启）

## 常见问题

### Q1: OpenCode 卡住了怎么办？
**A**:
1. 等待 10 分钟（可能在处理复杂任务）
2. 检查日志是否有重复操作
3. 如果确认卡住，杀掉进程：`process({ action: "kill", sessionId: "..." })`
4. 重启任务

### Q2: OpenCode 提前退出了怎么办？
**A**:
1. 检查 Git 状态（是否有提交）
2. 检查进度文件（是否更新）
3. 如果有提交且测试通过 → 任务完成
4. 如果没有提交或测试失败 → 重启任务

### Q3: OpenCode 做的任务不对怎么办？
**A**:
1. 检查进度文件（是否更新了错误的 Version）
2. 回滚 Git commit：`git reset --hard HEAD~1`
3. 更新 prompt，提供更清晰的指令
4. 重启任务

### Q4: 如何判断 OpenCode 是否偷懒？
**A**:
- ✅ 有文件改动（+X 行代码）
- ✅ 有 Git commit
- ✅ 测试通过
- ✅ 进度文件更新
- ❌ 如果以上都没有 → 偷懒了，重启
